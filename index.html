Status Definitions

Status is computed deterministically from the underlying data after each run — it is not Claude's judgment.

REDTriggered if any of: concern score ≥ 70, action is Review/Escalate, 2+ negative agency outlooks, YTD stock drop > 30%, or leverage > 5x AND FCF negative.
AMBERTriggered if any of: concern score ≥ 30, action is Watch, 1 negative outlook, leverage > 5x, FCF negative, YTD drop > 20%, or 1M drop > 15%.
GREENNone of the above triggers fired. Routine monitoring.

Action Tiers

MonitorStandard surveillance, no action required.
WatchIncreased scrutiny, near-term review possible.
ReviewFormal credit review warranted.
EscalateCommittee discussion / 2LOD challenge required.

Flag Count (Overview Tab)

The Flags column on the Overview tab shows the count of FLAGGED triggers from the 9 universal red flags evaluated per company. The progress bar weights flagged (2x) plus watch (1x) signals against a maximum of 18.

Format: X/9 (+Y~) where X = flagged count and Y = watch count.

See the Red Flags tab for the full per-company heatmap and the section below for flag definitions and thresholds.

Ratings & Outlook

  • M / S / F = Moody's / S&P / Fitch
  • Outlook colors: green = Positive, gray = Stable, red = Negative or RUR
  • ⚠ icon = rating action over 12 months old (stale)
  • Source: agency press releases & news synthesis via Claude web search, plus manual override file (ratings_override.json) for high-touch names

Market Data

  • Source: Yahoo Finance via yfinance library
  • Current price: latest available close
  • Market Cap: pulled from yfinance (price × shares outstanding); refreshes daily
  • Enterprise Value (EV) = Market Cap + Total Debt − Cash. Calculated row by row using yfinance market cap plus Claude-sourced debt/cash (Phase 2 will move debt/cash to SEC EDGAR).
  • 1D / 1M / YTD: calculated from closing prices
  • 52W high / low: trailing 52-week trading range
  • Refresh: daily

Macro Indicators

  • HY OAS / IG OAS: ICE BofA US Corporate index option-adjusted spreads
  • 10Y / 2Y UST: US Treasury closing yields
  • VIX: CBOE Volatility Index
  • S&P 500: index level & 1-day change
  • Nasdaq: Nasdaq Composite index (^IXIC) & 1-day change
  • Dow: Dow Jones Industrial Average (^DJI) & 1-day change
  • WTI / Brent: front-month crude oil futures (CL=F, BZ=F)
  • Gold: front-month gold futures (GC=F)
  • EUR/USD: spot FX
  • Sources: spreads, yields, VIX, S&P 500 level via Claude web search; equity indices (Nasdaq, Dow), commodities & FX via yfinance
  • Macro tab: 18 indicators pulled directly from FRED (Federal Reserve Economic Data) covering rates, spreads, inflation, labor, and activity. Updates daily. Cache TTL 20 hours.

Financials

  • Source: SEC EDGAR XBRL data, pulled directly from data.sec.gov. No paid feeds, no Claude estimates for the metrics below (for US 10-K filers and 20-F annual filers).
  • Refresh: Weekly — cache is refreshed if older than 6 days. Manual refresh via workflow input.
  • LTM construction: Sum of the 4 most recent non-overlapping quarterly facts. For 20-F annual filers, the latest annual figure is used (labeled).
  • Net Debt = Long-Term Debt + Short-Term Debt − Cash & Equivalents
  • EBITDA = Operating Income + Depreciation & Amortization (GAAP construction, no company-reported adjustments)
  • FCF = Operating Cash Flow − CapEx
  • ND/EBITDA = Net Debt / LTM EBITDA
  • Revenue YoY % = LTM Revenue vs. trailing 4-quarter sum from a year earlier
  • All dollar figures in $Bn
  • Each row carries a source tag: SEC = direct SEC EDGAR data, EST = Claude estimate (used for non-SEC filers)
  • A ⚠ icon on a row indicates data quality warnings — hover to see the specific issue (e.g., "Cash not found", "Revenue YoY = -55% verify")
  • Market Cap moved to the Market Data tab (it's a price-derived market metric, not a filing-derived financial)

SEC EDGAR Data Quality

  • Authoritative: XBRL-tagged GAAP data from actual filings. Same numbers as in the 10-K/10-Q.
  • Tag fallback chains: Companies tag concepts slightly differently. We try the primary tag (e.g., Revenues), then fall back to alternatives (e.g., RevenueFromContractWithCustomerExcludingAssessedTax) if missing.
  • Validation: Implausible values (cash > assets, ND/EBITDA outside ±50x, etc.) trigger warnings shown as ⚠ on the row.
  • 20-F filers (Toyota, BP, AB InBev): annual data only; quarterly fields show the most recent annual figure.
  • Non-SEC filers (Nissan, Hyundai, Imperial Brands): no SEC data available; financials fall back to Claude web-search estimates.
  • Cache file: financials_cache.json in the repo — auditable record of what data was used.

Refresh Cadence

  • Dashboard runs daily at 8:00 AM ET on weekdays
  • Daily refresh: Market data, commodities/FX, equity indices (yfinance); ratings, news, top 3 (Claude); FRED macro indicators (cache TTL 20 hours); status compute
  • Weekly refresh: SEC EDGAR financials (cache TTL = 6 days; refreshes on first run after expiry)
  • Manual override file (ratings_override.json) applies after auto-pull

Audit Trail

  • runs.json in the repo: rolling log of the last 60 runs, capturing data sources called, success/failure counts, validation warnings, output stats, and timing. Auto-trimmed.
  • financials_cache.json in the repo: snapshot of the SEC data used for the current run. Inspectable.
  • macro_cache.json in the repo: snapshot of the FRED data used for the current run. Inspectable.
  • Row-level provenance: each financial cell carries a source tag (SEC or EST) and a warning marker (⚠) where validation rules fired.
  • Validation rules: impossible values (cash > assets, ND/EBITDA > 50x, EBITDA margin > 80%, etc.) trigger row warnings. Hover to see the specific rule that fired.

Data Limitations

  • 20-F filers (Toyota, BP, Anheuser-Busch InBev) report annually rather than quarterly — financial data may be 6–12 months old
  • Non-SEC filers (Nissan, Hyundai, Imperial Brands) have limited financial data; market data still available via yfinance
  • News & key dev: best-effort synthesis from public sources; may miss developments outside the search window
  • This dashboard is a personal scanning tool, not a regulated system of record. Always verify against authoritative sources before relying on data for committee work.
